# 基于 Mobile OpenARM 的家庭出门物品准备助手系统设计

## 1. 项目背景与意义

服务机器人正在从实验室和工业场景进入家庭、养老、酒店、公寓和办公空间。家庭环境相比工厂更加开放：物品摆放不固定，光照和遮挡复杂，用户习惯差异明显，机器人还需要与人、家具和宠物安全共处。因此，一个真正有价值的家庭机器人系统不能只完成单个抓取动作，还要能够理解用户需求、规划任务流程、处理异常，并在不确定时向用户反馈。

本项目以 Mobile OpenARM 为设想平台，设计“家庭出门物品准备助手系统”。系统面向用户出门前的高频需求：寻找钥匙、钱包、手机、耳机、眼镜、药盒、雨伞、文件夹、电脑、证件等物品，并将它们集中放置到玄关托盘或整理区。同时，系统会根据模拟天气、交通、日程和用户习惯做克制的主动提醒，例如雨天建议携带雨伞，会议前建议携带电脑和文件，交通拥堵时提醒提前出门。

本项目的课程价值在于把机器人学中的感知、导航、机械臂操作、人机交互和系统工程思想组织成一个完整方案。项目不追求真实硬件部署，而是通过文档、图表、模拟数据和静态演示页面展示系统设计能力。

## 2. 系统总体目标

系统目标是形成“用户清单输入 - 系统主动建议 - 用户确认 - 机器人搜索与准备 - 状态反馈”的闭环流程。

用户可以输入一组出门物品。系统将清单结构化后，读取模拟环境数据和用户习惯数据，判断是否需要建议补充物品或出行提醒。用户确认最终清单后，系统根据家庭语义地图和物品常见位置生成搜索顺序，并在演示中模拟 Mobile OpenARM 完成导航、识别、抓取、放置和任务状态更新。

## 3. 家庭应用场景

本系统聚焦真实家庭，而不是校园或固定实验场地。典型场景包括：

- 普通上班出门：准备钥匙、钱包、手机、耳机。
- 雨天出门：系统根据天气建议携带雨伞。
- 会议出门：系统根据日程建议携带电脑、文件夹和证件。
- 老人就医出门：系统提醒携带药盒、眼镜、手机和证件。
- 交通拥堵：系统提醒用户提前出门，而不是替用户做最终决定。

## 4. 用户需求

用户最直接的需求是减少出门前遗漏物品和寻找物品的时间。对于上班族，遗漏钥匙、钱包或证件会造成明显时间成本；对于老人或行动不便人群，忘记药品、眼镜或手机可能带来安全风险；对于会议、就医等有明确目的的出门场景，系统还需要根据场景给出更合理的补充建议。

用户同时需要可解释和可控的系统行为。机器人不能擅自替用户决定所有事项，应采用“建议 - 确认 - 执行”的模式；当识别置信度低、路径受阻或抓取失败时，系统应暂停、切换策略或询问用户，而不是强行执行。

## 5. 核心功能

1. 清单输入与任务解析：将用户输入的中文物品清单转为结构化任务。
2. 主动建议：根据天气、交通、日程和习惯推荐补充物品或提醒。
3. 多模态识别设计：用视觉判断类别，用电子标签确认身份，用习惯模型预测位置。
4. 搜索顺序规划：根据家庭区域和物品常见位置生成机器人作业顺序。
5. 抓取与放置流程设计：根据物品类型选择模拟抓取策略，并统一放置到玄关整理区。
6. 状态反馈与异常处理：展示待处理、执行中、已完成、异常等任务状态。

## 6. 创新点

本项目的创新不在单一算法，而在家庭服务机器人的流程整合。系统将出门准备这类生活任务拆解为可解释的工程模块，并把外部环境信息、用户习惯、电子标签和视觉识别组合起来。电子标签不是视觉的替代品，而是辅助确认重要物品身份，降低遮挡、相似物品和误识别带来的风险。

项目还强调主动决策的边界。系统可以建议用户携带雨伞、外套或文件，但必须等待用户确认；在不确定目标时不抓取，在抓取失败时不强行重复操作，体现家庭场景下的安全优先原则。

## 7. 可行性分析

从课程项目角度看，本项目适合以系统设计和非实物演示方式完成。Mobile OpenARM、RGB-D 相机、电子标签读取、导航、识别和机械臂控制均作为模块边界进行设计说明，不要求真实接入。演示系统通过模拟 JSON 数据和前端页面展示任务决策、流程推进、日志反馈和架构关系，能够清楚表达工程逻辑。

从工程扩展角度看，项目结构保留了未来接入真实算法的接口边界：清单解析可替换为自然语言处理模块，识别模拟可替换为视觉模型输出，导航和抓取模拟可替换为 ROS 节点状态，环境数据可替换为真实 API。

## 8. 项目总结

本项目围绕家庭出门物品准备这一高频生活场景，构建了一个适合课程展示的机器人系统设计方案。它既体现 Mobile OpenARM 作为移动操作平台的能力边界，也展示了服务机器人在真实家庭场景中需要面对的任务理解、主动提醒、多模态识别、安全约束和异常处理问题。最终成果由项目文档、系统设计手册、流程图、架构图、模拟数据和静态演示页面组成，可用于课程提交和答辩演示。
